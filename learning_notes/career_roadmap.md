## üó∫Ô∏è The Roadmap: "Just-In-Time Fundamentals"
*We avoid "Tutorial Hell" by building real projects. We learn the theory only when the code breaks.*

### **Phase 1: The Foundation (Weeks 1-4)**
*   **Focus:** **Python Mastery & The CLI.**
*   **Project:** **The NexusDojo CLI (Branch A).**
*   **Skills:**
    *   **Core:** Python syntax, File I/O, Environment Variables, `pip`/`venv`.
    *   **AI:** Basic API integration (Gemini/OpenAI), System Prompting.
    *   **Goal:** Comfort in the terminal. Writing code that runs.

### **Phase 2: The Architect (Weeks 5-8)**
*   **Focus:** **Data, RAG, & MCP Basics.**
*   **Project:** **"Chat with my Notes" (Add-on to NexusDojo). Begin building a simple, local **MCP Server** to expose local data (like your learning notes) to an AI model.
*   **Skills:**
    *   **Data:** JSON, Embeddings (Vector DBs like Chroma/Pinecone).
    *   **AI:** Naive RAG pipelines, Chunking strategies. Intro to MCP concepts and API.
    *   **Goal:** Understanding how to feed "Context" to the AI, and how to standardize that access.

### **Phase 3: The Engineer (Weeks 9-12)**
*   **Focus:** **Agents, Logic, & Advanced MCP.**
*   **Project:** **"Auto-Researcher Agent" leveraging your MCP Server.**
*   **Skills:**
    *   **Logic:** Loops, Control Flow, Error Handling (`try/except`).
    *   **AI:** Tool Use (Function Calling), Agentic Loops, LangChain (intro). **Advanced MCP Server development and integration.**
    *   **Goal:** Building systems that "think" and "act," with standardized, secure data access.

### **Phase 4: The Professional (Weeks 13-16)**
*   **Focus:** **Shipping & Portfolio.**
*   **Project:** **Web UI (Streamlit/FastAPI) + Portfolio Site.**
*   **Skills:**
    *   **Web:** Basic frontend/backend connection.
    *   **Ops:** Git, Deployment (Vercel/Render).
    *   **Career:** Polish the Resume/LinkedIn with the "Solutions Engineer" title, **highlighting MCP expertise.**

## üß† The Learning Protocol: "Active Vibe Coding"
1.  **Vibe Code:** Use AI to generate the initial prototype (Speed).
2.  **The Stop Sign:** Before running critical code, **STOP**.
3.  **The Audit:** Ask: "Explain the control flow. Why did you use this library? Where is the security risk?"
4.  **The Result:** You never execute code you cannot explain.
